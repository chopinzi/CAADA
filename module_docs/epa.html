EPA (CEMS) package API¶

This package focuses on downloading and reading US EPA Continuous Emissions Monitoring System (CEMS) .csv files. The web module has a class available to download the hourly or daily CEMS files from the EPA FTP server. The readers module helps read in these .csv files.

Module: web¶

class caada.epa_cems.web.EPAFTP(url: str = 'newftp.epa.gov', *args, **kwargs)¶

Class that manages a connection to the EPA FTP server.

This wraps the standard ftplib.FTP class but supplies the URLs by default and contains a helper download() method to support batch downloading of Continuous Emission Monitoring System (CEMS) files.

This can be used as a context manager:

with EPAFTP() as ftp:
    ftp.download(...)

This is encouraged because it ensures that the FTP connection is closed whether or not an error occurs.

download(time_res: str, start_time: Union[datetime.datetime, pandas._libs.tslibs.timestamps.Timestamp], stop_time: Union[datetime.datetime, pandas._libs.tslibs.timestamps.Timestamp], save_dir: Union[str, pathlib.Path], states: str = 'all', unzip: bool = True, delete_zip: bool = True)¶

Download a collection of EPA CEMS files.

Parameters

  • time_res – Which time resolution of files to get. Options are “daily” or “hourly”.

  • start_time – Beginning and end of the time period to download data for. See Notes, below. This may be any format that pandas.Timestamp recognizes as a valid input.

  • stop_time – Beginning and end of the time period to download data for. See Notes, below. This may be any format that pandas.Timestamp recognizes as a valid input.

  • save_dir – Path to save the CEMS files to.

  • states – Which states to download data for. The string “all” will download all states, and “conus” will download only continental US states. If you want to limit to specific states, pass a sequence of state abbreviations, e.g. (“ca”, “or”, “wa”).

  • unzip – Whether to unzip the .zip files after downloading.

  • delete_zip – Whether to delete the .zip files after extracting the contained .csv file. Has no effect if unzip is False.

Notes

Time filtering is done based on the start time of each file. Since the hourly data is organized into monthly files and the daily data in organized into quarterly files, a time range from 2020-03-15 to 2020-04-15 would download the April 2020 hourly file or Q2 daily file (because April 1, 2020 is the start date for both files), however it will not download the March 2020 hourly or Q1 2020 daily file, even though the first part of the date range overlaps those files.
